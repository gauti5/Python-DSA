## Linked List

-> Linked list is type of datastructures used to store elements in a linear order. unlike an array,
which stores elements in continous memory locations, a linked list stores elements at random memory
location and connects them using pointers.

each element in a linked list is called a node, and each node contains:
Data (the actual value)
Pointer (or link) to the next node in the list.

Types of linked list:

1. Singly linked list
each node has:
Data
pointer to the next node

2. Doubly Linked List
Each node has:
Data 
pointer to the next node
pointer to the previous node

3. Circular Linked List
In Circular Singly linked List, Last Node points backs to the first node
In Circular doubly linked List, both last and first nodes are linked to each other in both directions


## Stack :

-> Stack follows LIFO (last in first out)
-> A stock works like stack of plates in your kitchen  (The last plate you place on top is the first one you pick)

# Main Operations

Push -> add an element to the top -> O(1)
pop -> remove the top elements -> O(1)
peek/top -> view the top element without removing it -> O(1)
isEmpty -> check if the stack is empty or not -> O(1)


## Queue : 

Queue follows FIFO [First in First Out], this means the first element added to the queue will be the first one to be removed.

Example : Just like people standing in a line for cinema ticket

# Main Operations :

Enqueue [add] -> add an element to the back of queue
Dequeue [remove] -> remove an element from the front of the queue.
Front [peek] : View the element at the front without removing it
isEmpty : check if the queue is empty
